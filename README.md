# Parallel Example: Motion Continuity Lab

This packet models how an animation-focused AI project could become a research journal and paper-lite draft. It is adjacent to animation and tool-switching work, but it does not describe the student's actual project.

## Core idea

The imagined Space is a **Motion Continuity Lab**. A user writes a short animation scene, then asks AI tools to turn it into frame-by-frame beats or scene directions. The project asks whether the output preserves motion continuity or just produces a set of attractive but disconnected moments.

## Research question

Which AI tools are best at helping with animation scene generation when the real challenge is motion continuity instead of a single pretty frame?

## Space idea

Inputs:

- Character description
- Three-beat action: start, change, end
- Mood shift
- Tool mode: storyboard beats, frame prompts, or animator notes

Outputs:

- Sequence of scene beats
- Continuity checklist
- Notes on what changes, what stays consistent, and what breaks
- Reflection prompt asking whether the result feels like animation or a slideshow

## What this packet demonstrates

- How to turn an animation interest into a testable continuity question
- How to evaluate sequences instead of single outputs
- How to connect classroom testing to text-to-video and temporal-consistency research
- How to write about creative AI tools without overclaiming
